You are an expert in AI agent trajectory analysis.

## Task Description
You will be given two spreadsheet problem-solving trajectories produced by:
- <raw_agent>: an agent without skills that fails the task
- <agent_with_skill>: the same agent augmented with a skill file, which successfully completes the task

Inside the agent trajectories:
- The system prompt defines the agent's action pattern.
- Assistant messages contain the agent's reasoning and actions.
- User messages contain observations returned by the environment.

Your task is to analyze how adding the skill enables a previously failing agent to succeed, and to extract a small set of distinct **Success Cause Items** that explain why the improvement occurs.

A Success Cause Item must describe a systematic and causal reason why the skill leads to success, such as:
- improving planning or tool usage,
- correcting incorrect assumptions,
- expanding the agent’s search space,
- enforcing better decomposition of the task,
- or stabilizing previously error-prone behavior.

## Important Rules
1. **Reasoning first**:  
   You must first analyze why <raw_agent> fails and why <agent_with_skill> succeeds, and only then summarize insights as Success Cause Items.
2. **Causality**:  
   Only extract success causes that can be reasonably attributed to guidance in the skill file.  
   Do NOT include improvements that would likely occur without the skill.
3. **Contrastive grounding**:  
   Each success cause must be supported by a contrast between <raw_agent> (failed) and <agent_with_skill> (successful).
4. **Non-redundancy**:  
   Do NOT produce overlapping or semantically duplicate items.
5. **Generality**:  
   Each success cause must be abstract enough to generalize to other spreadsheet tasks.
6. **Mechanism-focused**:  
   Each item must explain *how* the skill changes the agent’s behavior in a way that leads to success, not just *what* improved.
7. **Brevity constraint**:  
   You may extract **at most 3 Success Cause Items**. So only extract the most significant and generalizable items.


## Output Format (STRICT)

Your output must strictly follow this Markdown template for each item:

```
# Success Cause Item i  
## Title  
<Short descriptive title of the success cause>

## Description  
<One-sentence summary of the success cause>

## Content  
<1–3 sentences explaining:
- how the behavior differs between the two agents, and  
- which guidance in the skill file leads to this success>
```
